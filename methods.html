HTTP METHODS

    • REQUEST METHODS
        • GET

          Transfer current selected representation of the resource identified by the Request URL. Returned resource to message body of the response as an entity.

          Most commonly used HTTP Method.

          Most supported by all complient.

        • HEAD

          Same as GET method except the entity is not included in th response.

          Use to Retrieve meta data.

          Like GET, must be supported by all general purpose.

        • PUT

          Store the enclosed entity in the message body under the specified Request.

        • DELETE

          Remove the associated entity.

        • OPTIONS
        • TRACE

          It will echo the message back to the client. Whatever it recieves it sends back to the client.

        • CONNECT

          Use when connecting HTTPS resources. It establishes encyption tunnel.

    • EXTENSION METHODS

      WebDAV(RFC 4918)
      PROPFIND , PROPPATCH , MKCOL , COPY , MOVE , LOCK , UNLOCK

    • SAFE METHODS

      Not expecting changes.
      (GET, HEAD, OPTIONS, TRACE)

    • IDEMPOTENT METHODS

      If you use it again you will get the same result.
      (GET, HEAD, PUT, DELETE)

    • CACHEABLE METHODS

      The Request that you get can be cache so that if you can request again it will direct to the cache.
