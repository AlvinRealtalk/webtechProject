  • What is HTTP?


    The Hypertext Transfer Protocol (HTTP) is an application protocol for distributed, collaborative, and hypermedia information systems. HTTP is the foundation of data communication for the World Wide Web. Hypertext is structured text that uses logical links (hyperlinks) between nodes containing text.

  • HTTP is a Stateless Protocol


    HTTP is called a stateless protocol because each command is executed independently, without any knowledge of the commands that came before it. This is the main reason that it is difficult to implement Web sites that react intelligently to user input. This shortcoming of HTTP is being addressed in a number of new technologies, including ActiveX, Java, JavaScript and cookies.

  • Protocol


    • • HTTP typically runs on top of TCP/IP, using TCP port 80 by default (TCP port 443 for HTTPS).

    • • HTTP resources are identified using URIs (specifically, HTTP URLs)
      1. o Scheme (http: or https:)
      2. o (optional) authentication information o Host and (optional) port number
      3. o Path (resolved to the document root on the server) to the resource o (optional) scheme-specific parameters
      4. o (optional) URL-encoded query
      5. o (optional) bookmark (or fragment identifier)

    • • HTTP is based on client-server architecture

    • • Clients, aka user agents (UA):
      1. o Web browsers, web crawlers, email clients, other end user tools and applications

    • • Servers:
      1. o Origin servers, proxy servers, gateways, tunnels

    • • HTTP uses a request-response standard protocol
      1. o The client sends an HTTP request message to the server
      2. o The server processes the request and replies with an HTTP response message

    • • HTTP is a stateless communications protocol
      1. o Servers do not keep information about clients in between requests
      2. o Web applications effect session tracking using mechanism such as cookies on URL-encoded session information to keep track of related client requests

    • • HTTP provides support for other functionalities such as cache control, content media type (MIME) specification, language and character set specification, content/transfer coding, client-server protocol negotiations, persistent connections, request pipelining, etc.
